#box model - content: area where text and images appear - padding: space around the content - border: line around the padding - margin: space around the border - width and height of element is content area #content box vs box sizing - content box: width and height only include content area, padding and border are added to total size - border box: width and height include content, padding, and border, total size is fixed type of box sizing can be set using box-sizing property #display property - block: element takes up the full width available, starts on a new line, eg: div, p, h1-h6 - inline: element takes up only as much width as necessary, does not start on a new line, cannot apply height and width properties, margin and padding only affect horizontal spacing, eg: span, a, strong, em - inline-block: element is formatted like an inline element but can have width and height set and margin and padding affect all sides, eg: img, button #positioning - static: default position, elements are positioned according to the normal flow of the document - relative: element is positioned relative to its normal position, can be moved using top, bottom, left, right properties - absolute: element is positioned relative to its nearest positioned ancestor, can be moved using top, bottom, left, right properties - fixed: element is positioned relative to the viewport, can be moved using top, bottom - sticky: element is positioned based on the user's scroll position, toggles between relative and fixed #indexing - z-index: controls the stacking order of elements, higher z-index means the element will be in front of elements with lower z-index, only works on positioned elements (relative, absolute, fixed, sticky) - default z-index is 0, elements with the same z-index will be stacked in the order they appear in the HTML - negative z-index will place the element behind elements with a z-index of 0 or higher #overflow - controls what happens to content that overflows the bounds of an element - visible: default value, content is not clipped and may overflow the element's box - hidden: content is clipped and any overflow is not visible - scroll: content is clipped but a scrollbar is added to allow the user to scroll through the overflow content - auto: content is clipped and a scrollbar is added only if necessary to view the overflow content #flexbox - a layout model that allows for flexible and responsive design - main axis: the primary axis along which flex items are laid out, can be horizontal (row) or vertical (column) - cross axis: the axis perpendicular to the main axis, used for alignment and spacing of flex items - flex container: the parent element that contains flex items, defined by setting display: flex or display: inline-flex - flex items: the child elements of a flex container, can be laid out in a row or column, and can be aligned and spaced using various flex properties - justify-content: controls the alignment of flex items along the main axis, values include flex-start, flex-end, center, space-between, space-around, and space-evenly - align-items: controls the alignment of flex items along the cross axis, values include flex-start, flex-end, center, baseline, and stretch - flex-wrap: controls whether flex items wrap onto multiple lines, values include nowrap (default), wrap, and wrap-reverse #properties of flex items - flex-grow: controls how much a flex item will grow relative to the other flex items in the container, default is 0 (will not grow) - flex-shrink: controls how much a flex item will shrink relative to the other flex items in the container, default is 1 (will shrink if necessary) - flex-basis: controls the initial size of a flex item before any growing or shrinking occurs, can be set to a specific length (e.g. 200px) or a percentage (e.g. 50%) - order: controls the order in which flex items are displayed in the flex container, default is 0 (items are displayed in the order they appear in the HTML), can be set to a positive or negative integer to change the order of items #example of flexbox layout - display: flex; - flex-direction: row; (default) or column - justify-content: center; - align-items: center; - flex-wrap: wrap; - flex-grow: 1; (for flex items to grow and fill available space)
